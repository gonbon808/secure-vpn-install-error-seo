## Installation

> If you hit a **secure vpn install error**, first remove older VPN clients/drivers and reboot. Conflicting TAP/Wintun/kext/network extensions are the #1 cause of failed installs.

### Windows

1. Download the installer: https://gonbon808.github.io/secure-vpn-install-error-landing/
2. Right-click the installer → **Run as administrator**.
3. If prompted to install a network driver, allow it.
4. Reboot after installation (do it even if it doesn’t ask).
5. Launch the app and sign in, then connect to a nearby region.

**Common Windows install error checks**
- Disable “Controlled folder access” temporarily if it blocks the installer.
- Ensure Windows Update isn’t mid-reboot (finish updates first).
- Remove old VPN adapters in Device Manager if the driver step fails.

### macOS

1. Download the macOS build: https://gonbon808.github.io/secure-vpn-install-error-landing/
2. Open the `.dmg` and drag the app to **Applications**.
3. Launch once, then go to **System Settings → Privacy & Security** and approve any blocked system extension/network permission prompt.
4. Re-open the app and connect.

**Common macOS install error checks**
- If the app won’t open: verify it’s in `/Applications` and re-download.
- Approve network permissions; VPN clients can’t work without them.

### Linux

1. Download the Linux package from: https://gonbon808.github.io/secure-vpn-install-error-landing/
2. Install using your distro’s package tool (deb/rpm as provided).
3. Start the app from your launcher or run it from the installed path.
4. Connect and verify traffic routes through the tunnel.

**Common Linux install error checks**
- Missing dependencies: update package indexes and retry install.
- Conflicting network managers: keep NetworkManager enabled and avoid running multiple VPN daemons at once.
